## 目标
- 参考“需求进度”页面的交互与样式，在订单详情页增加“更多”按钮与“删除订单”入口。
- “更多”按钮打开操作菜单：撤销订单、删除订单；删除需在订单已撤销后执行。

## 现状核对
- 订单详情逻辑：`pages/order/detail/detail.js:70-119` 已有 `onCancel`、`onDelete`、`onModify`、`onContact`，删除逻辑调用云函数 `orders_remove` 与 `requests_remove`。
- 订单详情 UI：`pages/order/detail/detail.wxml:40-45` 已有操作区，含“撤销”“修改订单”“联系客服”，以及在 `order.status==='已取消'` 时显示的“删除订单”按钮。
- 参考页“更多”按钮：`pages/request/progress/progress.wxml:39-41`，样式参见 `pages/request/progress/progress.wxss:24-27`，事件 `onMoreTap` 调用操作菜单并路由到撤销或删除。
- 状态值存在中英文混用：详情页 JS 使用 `'canceled'`，WXML 使用 `'已取消'` 条件，导致“删除订单”可能不显示。

## 改动方案
1. WXML：在订单详情操作区追加“更多”按钮
   - 在 `pages/order/detail/detail.wxml` 的 `<view class="section actions">` 内追加：
     - `<button class="btn ghost more" bindtap="onMoreTap"><image class="more-icon-img" src="/images/更多.svg" mode="aspectFit" /></button>`，放在操作区右侧，与参考页一致。
   - 修正“删除订单”显示条件以兼容中英文状态：`wx:if="{{order.status==='canceled' || order.status==='已取消'}}"`。
   - 同步“撤销”“修改订单”的 `disabled` 条件：包含 `order.status==='canceled'`。

2. WXSS：补充“更多”按钮样式
   - 在 `pages/order/detail/detail.wxss` 增加与参考页一致的样式：
     - `.btn.more{flex:0 0 65rpx;height:58rpx;padding:0;display:flex;align-items:center;justify-content:center;border-radius:36rpx;border:none;background:transparent}`
     - `.more-icon-img{width:70rpx;height:70rpx}`（与参考页的 `more-icon` 尺寸对齐，改为图片类名）。

3. JS：实现 `onMoreTap` 操作菜单
   - 在 `pages/order/detail/detail.js` 新增：
     - `onMoreTap(){ wx.showActionSheet({ itemList:['撤销订单','删除订单'], success:(res)=>{ if(res.tapIndex===0){ this.onCancel() } if(res.tapIndex===1){ this.onDelete() } } }) }`
   - 保持现有 `onDelete` 业务规则：仅当状态为撤销（`'canceled'` 或 `'已取消'`）时允许删除，否则弹出“请先撤销订单”。

4. 资源引用
- 复用参考页图标：`/images/更多.svg` 已存在（参考 `pages/request/progress/progress.wxml:39-41`）。无需新增资源。

## 验证步骤
- 从订单列表进入详情（参考 `pages/profile/orders/orders.js` 的 `wx.navigateTo`）。
- 场景一：未撤销订单
  - “更多”菜单显示两项；点击“删除订单”提示先撤销；点击“撤销”后状态更新为 `canceled` 并提示“已撤销”。
- 场景二：已撤销订单
  - 操作区显示“删除订单”按钮（兼容中英文状态）；“更多”菜单中“删除订单”可直接删除并返回列表。
- 检查样式与参考页一致，图标正常显示。
