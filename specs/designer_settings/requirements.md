# 需求文档：设计师个人中心设置功能

## 介绍

为设计师个人中心的「设置」模块开发后端云函数，覆盖隐私管理、账号与安全信息查询和退出登录三个功能域。消息通知设置已在 `designer_settings` 云函数中完成，本次在该云函数基础上扩展。

## 需求

### 需求 1 - 隐私设置管理

**User Story:** 作为设计师，我希望能持久化保存我的隐私偏好（公开作品集、允许咨询、展示评价），以便每次打开页面时保持我设置的状态。

#### 验收标准

1. 当设计师进入「隐私管理」页面时，系统应从云端读取该设计师的隐私设置并回显到开关控件上。
2. 当设计师切换任意隐私开关时，系统应在 1 秒内将新状态持久化到云数据库；若保存失败，系统应将开关回滚到修改前的状态并提示「保存失败」。
3. 当设计师为新用户、尚未有隐私设置记录时，系统应返回默认值：`publicPortfolio=true`、`allowConsult=true`、`showRating=false`。
4. 当调用方不是设计师身份（roles ≠ 2 且 roles ≠ 0）时，系统应返回 `NOT_DESIGNER` 错误。

---

### 需求 2 - 账号与安全信息查询

**User Story:** 作为设计师，我希望在「账号与安全」页面看到真实的手机号（脱敏）、微信绑定状态和实名认证状态，而非硬编码的 mock 数据。

#### 验收标准

1. 当设计师进入「账号与安全」页面时，系统应返回该设计师的手机号（保留前3位和后4位，中间用 `****` 遮掩）。
2. 系统应返回微信绑定状态：若 `_openid` 存在则为「已绑定」，否则为「未绑定」。
3. 系统应返回实名认证状态：若 `designers` 集合中 `realNameVerified=true` 则为「已认证」，否则为「未认证」。
4. 上述信息只读，本期不支持修改手机号或解绑微信。

---

### 需求 3 - 退出登录

**User Story:** 作为设计师，我希望退出登录时后端能同步失效我的会话，防止 token 被继续使用。

#### 验收标准

1. 当设计师确认退出登录时，系统应调用云函数清除该用户的服务端会话标记（`loginExpireAt` 设为当前时间，使旧 token 立即过期）。
2. 当云函数执行成功后，前端应清除本地 Storage，并跳转到启动页（splash）。
3. 当云函数调用失败时，前端仍应继续清除本地缓存并跳转，不因后端失败阻塞退出流程。
4. 无论用户角色（roles 0/1/2）均可调用退出登录接口。
