# 课程白名单授权功能 - 需求文档

## 一、需求背景

部分用户已在外部平台购买了灯光设计课程，需要在小程序中也能观看。目前只知道这些用户的手机号，且用户尚未在小程序中登录/注册过。希望实现：当这些用户在小程序中完成手机号授权登录后，自动获得课程观看权限。

## 二、业务约束

| 约束项 | 决策 |
|--------|------|
| 手机号获取方式 | 微信授权手机号（`getPhoneNumber`） |
| 授权课程范围 | 单一课程（灯光设计课程，courseId: `CO_DEFAULT_001`） |
| 授权有效期 | 永久有效 |
| 用户提示 | 静默激活，无提示 |
| 订单展示 | 创建订单，用户可在"我的订单"看到，无需特殊标识 |
| 外部订单号 | 不需要记录 |
| 导入方式 | 通过管理后台界面上传文件 |

## 三、课程信息

| 字段 | 值 |
|------|------|
| 数据库 `_id` | `da8988ea694a0a4108e867806b0c6371` |
| `courseId` | `CO_DEFAULT_001` |
| 课程名称 | 十年经验二哥 灯光设计课 |
| 当前价格 | ¥399 |

## 四、需求列表

### 需求 1 - 白名单数据导入

**用户故事**：作为管理员，我希望能够批量导入外部购买用户的手机号，以便这些用户在小程序登录后能自动获得课程权限。

#### 验收标准

1. When 管理员在管理后台上传包含手机号的 Excel/CSV 文件时，the 系统 shall 校验手机号格式（11位数字）并过滤无效记录。

2. When 导入的手机号在白名单中已存在时，the 系统 shall 跳过该记录并计入"重复跳过"统计。

3. When 导入完成后，the 系统 shall 显示导入结果统计（成功数、重复跳过数、失败数）。

4. While 白名单记录已创建，the 记录状态 shall 初始化为 `pending`（待激活）。

5. When 上传的文件格式不正确时，the 系统 shall 提示用户上传正确格式的文件（支持 .xlsx, .xls, .csv）。

---

### 需求 2 - 用户登录自动激活

**用户故事**：作为外部购买用户，我希望在小程序中授权手机号登录后，无需额外操作即可观看已购买的课程。

#### 验收标准

1. When 用户完成微信手机号授权时，the 系统 shall 检查该手机号是否存在于 `course_whitelist` 白名单中。

2. When 匹配到待激活的白名单记录时，the 系统 shall 静默为该用户创建一条课程订单（状态为 `completed`，来源标记为 `whitelist`）。

3. When 订单创建成功后，the 系统 shall 更新白名单记录状态为 `activated`，并记录激活时间和用户 OPENID。

4. When 激活过程中发生异常时，the 系统 shall 记录错误日志，但不影响用户正常登录流程。

5. While 用户已被激活过（白名单状态为 `activated`），When 用户再次授权手机号时，the 系统 shall 跳过激活流程，不重复创建订单。

---

### 需求 3 - 课程购买状态检查兼容

**用户故事**：作为白名单激活用户，我希望在课程详情页看到"已购买"状态，并能正常观看课程视频。

#### 验收标准

1. When 白名单用户访问课程详情页时，the `course_purchase_check` 云函数 shall 返回 `isPurchased: true`。

2. When 白名单用户点击"开始学习"按钮时，the 系统 shall 正常跳转到视频播放页。

3. When 白名单用户访问"我的订单"页面时，the 订单列表 shall 显示该课程订单（与普通购买订单显示一致）。

---

### 需求 4 - 白名单管理查询

**用户故事**：作为管理员，我希望能够查看白名单的激活情况，了解有多少用户已经激活。

#### 验收标准

1. When 管理员访问白名单管理页面时，the 系统 shall 显示白名单列表（支持分页）。

2. When 管理员按状态筛选时，the 系统 shall 支持筛选"全部/待激活/已激活"。

3. When 管理员查看统计数据时，the 系统 shall 显示总数、待激活数、已激活数及激活率。

4. When 管理员搜索手机号时，the 系统 shall 支持模糊搜索功能。

---

## 五、数据结构设计

### 5.1 新增集合：`course_whitelist`

| 字段 | 类型 | 必填 | 说明 |
|------|------|------|------|
| `_id` | String | 是 | 记录ID（自动生成） |
| `phone` | String | 是 | 用户手机号（11位） |
| `courseId` | String | 是 | 授权的课程ID（固定为 `CO_DEFAULT_001`） |
| `status` | String | 是 | 状态：`pending`（待激活）/ `activated`（已激活） |
| `source` | String | 否 | 来源/批次备注（如"2025年1月批次"） |
| `activatedAt` | Number | 否 | 激活时间戳 |
| `activatedUserId` | String | 否 | 激活用户的 OPENID |
| `orderId` | String | 否 | 创建的订单ID |
| `createdAt` | Number | 是 | 创建时间戳 |
| `createdBy` | String | 否 | 创建人（管理员ID） |

### 5.2 索引设计

| 索引字段 | 类型 | 说明 |
|----------|------|------|
| `phone` + `courseId` | 唯一索引 | 防止同一手机号重复导入同一课程 |
| `status` | 普通索引 | 便于按状态筛选 |
| `activatedUserId` | 普通索引 | 便于查询用户激活记录 |

### 5.3 订单标记字段

白名单激活创建的订单需要增加内部标记（用于区分来源，但前端不显示）：

| 字段 | 值 | 说明 |
|------|------|------|
| `source` | `"whitelist"` | 标记订单来源为白名单（内部使用） |
| `whitelistId` | 白名单记录 `_id` | 关联白名单记录 |

## 六、非功能性需求

1. **性能**：白名单检查应在用户登录流程中同步完成，不影响登录体验（< 500ms）。

2. **幂等性**：同一手机号多次授权登录，只创建一次订单。

3. **数据安全**：手机号属于敏感信息，管理后台访问需要管理员权限验证。

4. **容错性**：白名单激活失败不应影响用户正常登录流程。
