# 需求文档 — 设计师端「我的项目」功能完善

## 背景

设计师端「我的项目」页面（`designer-projects`）目前存在以下问题：

1. **WXML 全部硬编码** — 项目卡片为写死的3张静态假数据，未使用 `wx:for` 渲染 `projects` 数据
2. **云函数 `enrichProject` 缺少 `params` 展平** — 业主发布需求时，`space`/`budget`/`area`/`service`/`stage` 等字段嵌套在 `params` 对象中，前端读取顶层字段为空
3. **状态筛选不完整** — `ongoing`（进行中）仅映射 `review`，实际应包含 `review` + `design`
4. **搜索功能为空壳** — 搜索输入框无绑定事件
5. **无项目详情页** — "查看详情"按钮无跳转目标
6. **缺少交互状态** — 无 loading、空状态、下拉刷新
7. **缺少进度计算** — 卡片进度条需要根据 `steps` 数组动态计算百分比
8. **缺少相对时间** — 更新时间应显示"X分钟前"等友好格式

## 需求

### 需求 1 — 云函数 enrichProject 数据展平与丰富

**用户故事：** 作为设计师，我希望我的项目列表能正确显示项目名称、预算、面积、进度等信息，而不是空白。

#### 验收标准

1. 当 `requests` 集合文档中 `params` 字段存在时，`enrichProject` 函数应将 `params.space`、`params.budget`、`params.area`、`params.service`、`params.stage` 展平到文档顶层。
2. 当 `stage` 字段为 `'publish'` 且 `params.stage` 存在时，系统应使用 `params.stage` 覆盖顶层 `stage`。
3. 系统应根据 `space` 字段自动生成 `title`（格式：`{space}灯光设计需求`）。
4. 系统应根据 `steps` 数组计算 `progress`（完成步骤数 / 总步骤数 × 100）和 `currentStepText`（当前进行步骤名称）。
5. 系统应根据 `updatedAt` 或 `createdAt` 生成 `timeText`（刚刚 / X分钟前 / X小时前 / X天前 / 具体日期）。
6. 系统应根据 `status` 生成对应的 `statusTag`（颜色类名）和 `statusIcon`（图标名）。
7. 系统应从 `userNickname`/`userPhone` 拼接 `clientInfo` 客户信息。

### 需求 2 — 状态筛选修复

**用户故事：** 作为设计师，我希望点击"进行中"标签能看到所有正在进行的项目，包括"审核中"和"设计中"两种状态。

#### 验收标准

1. 当设计师选择"进行中"标签时，系统应查询 `status` 为 `review` 或 `design` 的项目。
2. 当设计师选择"待确认"标签时，系统应查询 `status` 为 `pending` 的项目。
3. 当设计师选择"已完成"标签时，系统应查询 `status` 为 `done` 的项目。

### 需求 3 — 前端项目列表页动态渲染

**用户故事：** 作为设计师，我希望项目列表显示我真实接手的项目，而不是静态假数据。

#### 验收标准

1. WXML 应使用 `wx:for="{{projects}}"` 循环渲染项目卡片。
2. 项目卡片应显示：标题、客户信息、状态标签（颜色随状态变化）、当前阶段、进度条（百分比动态）、更新时间。
3. 当项目列表为空时，系统应显示空状态提示（图标 + 文案）。
4. 页面应支持 loading 状态展示。
5. "查看详情"按钮应绑定事件，跳转到项目详情页。
6. 页面应支持下拉刷新。

### 需求 4 — 搜索功能

**用户故事：** 作为设计师，当项目较多时，我希望通过搜索关键词快速找到目标项目。

#### 验收标准

1. 当设计师在搜索框输入关键词时，系统应在本地对 `projects` 列表进行过滤（按 `title`、`clientInfo` 模糊匹配）。
2. 当搜索框清空时，系统应恢复显示完整列表。

### 需求 5 — 项目详情页（新建页面）

**用户故事：** 作为设计师，我希望点击项目卡片后能看到项目的完整信息，包括客户联系方式、工作流步骤和操作按钮。

#### 验收标准

1. 系统应新建 `designer-project-detail` 页面。
2. 页面应调用 `designer_projects` 云函数 `detail` action 加载项目数据。
3. 页面应展示：项目标题、状态标签、客户信息（昵称、电话、可拨打）、空间/面积/预算/服务类型、项目进度（工作流步骤列表 + 进度条）。
4. 设计师应能标记/取消标记步骤完成状态（调用 `update_step` action）。
5. 当所有步骤完成时，系统应自动更新项目状态为"已完成"并显示完成提示。
6. 页面底部应有"联系客户"按钮（调起手机拨号）。
