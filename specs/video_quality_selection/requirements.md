# 视频多分辨率选择功能 - 需求文档

## 介绍

为光乙共创平台的课程视频播放器添加多分辨率选择功能，让用户可以根据自己的网络状况和观看需求选择不同清晰度的视频（480p 标清、720p 高清、1080p 超清），同时在后台管理系统中支持多分辨率视频的上传和管理。

**背景：** 当前视频播放器仅支持单一分辨率，用户在网络较慢时无法降低清晰度，导致视频卡顿影响学习体验。管理员已经将视频的多个分辨率版本上传至云存储，需要前端和后台系统支持多分辨率的管理和播放。

**目标：**
- 提升用户观看体验，适应不同网络环境
- 提供智能的分辨率自适应能力
- 简化管理员的视频管理流程

## 需求

### 需求 2.1.1 - 分辨率选择 UI

**用户故事：** 作为学习者，我希望在观看视频时能够看到并选择不同的清晰度选项，以便在网络较慢时选择较低分辨率，或在网络良好时享受高清画质。

#### 验收标准

1. When 用户点击视频播放页右上角的"更多"按钮（三个点）时，系统应当展开播放选项弹窗。
2. The 系统 shall 在播放选项弹窗中显示"清晰度"选项区域（位于"倍速"选项下方）。
3. The 系统 shall 显示当前课时支持的所有分辨率选项，格式为"480p 标清"、"720p 高清"、"1080p 超清"。
4. The 系统 shall 以绿色背景高亮显示当前正在使用的分辨率。
5. When 某个分辨率的视频 URL 不存在时，系统应当禁用该选项并显示为灰色半透明状态。
6. The 系统 shall 采用与倍速选项相同的 UI 设计风格，保持界面一致性。

---

### 需求 2.1.2 - 分辨率切换功能

**用户故事：** 作为学习者，我希望能够在观看视频过程中随时切换清晰度，系统能够平滑过渡到新的分辨率，并继续从当前位置播放。

#### 验收标准

1. When 用户点击不同的分辨率选项时，系统应当立即记录当前视频的播放时间点（精确到秒）。
2. The 系统 shall 暂停当前视频播放。
3. The 系统 shall 更换视频源 URL 为选定分辨率对应的 URL。
4. The 系统 shall 在新视频加载完成后，自动跳转到之前记录的播放时间点。
5. The 系统 shall 自动恢复播放（如果之前是播放状态）。
6. The 系统 shall 显示 Toast 提示"已切换到 [分辨率名称]"（例如："已切换到 720p 高清"），持续 1.5 秒。
7. The 系统 shall 自动关闭播放选项弹窗。
8. When 切换失败时，系统应当显示错误提示"切换失败，请重试"，并保持原分辨率播放。

---

### 需求 2.1.3 - 默认分辨率策略

**用户故事：** 作为学习者，我希望系统能够记住我的清晰度偏好，下次观看视频时自动使用我上次选择的分辨率，避免每次都要重新设置。

#### 验收标准

1. When 用户首次使用小程序观看视频时，系统应当默认选择 1080p 超清分辨率。
2. When 用户手动切换分辨率后，系统应当将该分辨率偏好保存到本地存储（wx.setStorageSync）。
3. When 用户再次打开任何视频时，系统应当读取本地存储的分辨率偏好，并优先使用该分辨率。
4. While 用户偏好的分辨率在当前课时中不可用时，系统应当自动选择最高可用分辨率。
5. The 系统 shall 在控制台输出日志，记录默认分辨率选择的决策过程（用户偏好/降级）。

**示例：**
- 用户在视频 A 中选择 480p → 保存偏好
- 用户打开视频 B → 自动使用 480p
- 用户打开视频 C（仅有 720p 和 1080p）→ 自动使用 720p（降级）

---

### 需求 2.1.4 - 网络自适应切换

**用户故事：** 作为学习者，我希望在网络变慢导致视频持续缓冲时，系统能够自动降低清晰度，避免卡顿影响学习体验。

#### 验收标准

1. The 系统 shall 监听视频的 `waiting` 事件（缓冲状态）。
2. When 视频播放过程中出现持续缓冲（累计超过 20 秒）时，系统应当检查是否可以降低分辨率。
3. While 当前分辨率已经是最低分辨率（480p）时，系统应当不执行自动降级，仅显示"网络较慢"提示。
4. The 系统 shall 自动切换到下一级较低的分辨率（1080p → 720p → 480p）。
5. The 系统 shall 显示 Toast 提示"网络较慢，已自动切换到 [分辨率名称] 以保证流畅播放"，持续 2 秒。
6. The 系统 shall 记录当前播放时间点并跳转到新视频的相同位置。
7. The 系统 shall 重置缓冲计时器，继续监控后续播放情况。
8. When 网络恢复正常时，系统应当保持当前分辨率，不自动升级（避免频繁切换）。
9. The 用户 shall 仍然可以通过手动方式切换回更高分辨率。

**降级规则：**
- 1080p 超清 → 720p 高清
- 720p 高清 → 480p 标清
- 480p 标清 → 无法降级（显示提示）

### 需求 2.2.1 - 多分辨率视频上传（后台管理系统）

**用户故事：** 作为管理员，我希望在添加或编辑课程时，能够为每个课时上传多个不同分辨率的视频文件，方便管理视频资源。

#### 验收标准

1. When 管理员在课程编辑页面添加或编辑视频类型的课时时，系统应当显示"视频上传"区域。
2. The 系统 shall 提供三个独立的视频上传组件，分别标记为"480p 标清"、"720p 高清"、"1080p 超清"。
3. The 系统 shall 允许管理员通过以下两种方式之一添加视频：
   - 从本地上传视频文件到云存储
   - 直接输入云存储 URL（cloud:// 格式）
4. The 系统 shall 显示每个分辨率的上传状态：
   - 未上传：显示上传按钮或输入框
   - 上传中：显示进度条和百分比
   - 已上传：显示视频文件名、大小和"删除"/"替换"按钮
5. The 系统 shall 允许管理员删除已上传的某个分辨率视频。
6. The 系统 shall 允许管理员替换已上传的某个分辨率视频。
7. While 保存课时数据时，系统应当验证至少有一个分辨率的视频已上传，否则显示错误提示"请至少上传一个分辨率的视频"。
8. The 系统 shall 将视频 URL 保存为 `videoUrls` 对象格式：
   ```javascript
   {
     videoUrls: {
       "480p": "cloud://xxx-480p.mp4",
       "720p": "cloud://xxx-720p.mp4",
       "1080p": "cloud://xxx-1080p.mp4"
     }
   }
   ```

### 需求 2.2.2 - 分辨率标识展示（后台管理系统）

**用户故事：** 作为管理员，我希望在课程列表和详情页中能够快速看到每个课时支持哪些分辨率，方便我了解视频资源的完整性。

#### 验收标准

1. The 系统 shall 在课时列表的每个课时卡片上显示分辨率标签。
2. When 某个分辨率的视频存在时，系统应当显示实心绿色边框的标签（例如："480p"）。
3. When 某个分辨率的视频不存在时，系统应当显示灰色虚线边框的标签（例如："480p"）。
4. The 系统 shall 按顺序显示三个标签：480p、720p、1080p。
5. The 系统 shall 在课时编辑表单的顶部显示当前分辨率上传状态概览，形如："已上传：720p, 1080p | 未上传：480p"。

### 需求 2.3.1 - 课时数据模型变更

**用户故事：** 作为系统，我需要调整课时数据结构以支持多分辨率视频存储，同时保持对旧数据的兼容性。

#### 验收标准

1. The 系统 shall 将课时数据的 `videoUrl` 字段（字符串）改为 `videoUrls` 对象。
2. The `videoUrls` 对象 shall 包含三个可选键：`480p`、`720p`、`1080p`，值为对应的云存储 URL。
3. The 系统 shall 兼容旧的单 `videoUrl` 字段数据结构：
   - 当课时数据仅包含 `videoUrl` 时，系统应当将其视为 720p 视频
   - 小程序端可正常播放旧数据
4. The 云函数 `course_videos` shall 自动检测课时数据格式（新/旧），并适配处理。
5. The 云函数 `course_detail` shall 在返回课程详情时，保持数据格式的一致性。

**新数据结构示例：**
```javascript
{
  id: "1.1",
  title: "灯光设计基础",
  type: "video",
  videoUrls: {
    "480p": "cloud://xxx-480p.mp4",
    "720p": "cloud://xxx-720p.mp4",
    "1080p": "cloud://xxx-1080p.mp4"
  },
  duration: "01:01:51"
}
```

**旧数据结构（兼容）：**
```javascript
{
  id: "1.1",
  title: "灯光设计基础",
  type: "video",
  videoUrl: "cloud://xxx.mp4",  // 视为 720p
  duration: "01:01:51"
}
```

## 非功能性需求

### 性能要求
- 分辨率切换响应时间：< 2 秒（包括视频加载时间）
- 网络检测逻辑：不应占用主线程，不影响视频播放流畅度
- 本地存储读写延迟：< 100ms
- 云函数响应时间：< 1 秒

### 兼容性要求
- 必须完全兼容现有的单 `videoUrl` 课时数据（不破坏旧数据）
- 云函数必须同时支持新旧数据格式
- 后台管理系统应能处理混合数据（部分课时是新格式，部分是旧格式）

### 用户体验要求
- 分辨率切换时必须保持播放位置不变（误差 < 1 秒）
- 提供清晰的状态反馈（加载中、切换成功、切换失败）
- 网络自动降级时给予用户明确提示，避免困惑
- UI 设计遵循苹果官方风格，保持整体一致性

### 安全性要求
- 视频 URL 必须使用云存储临时链接，避免直接暴露永久 URL
- 云函数必须验证用户购买状态，未购买用户不得获取视频 URL
- 后台管理系统仅管理员可访问

## 验收测试场景

### 场景 1：手动切换分辨率（正常流程）

**前置条件：** 用户已登录，已购买课程，视频支持三种分辨率

**步骤：**
1. 用户进入课程视频播放页
2. 视频自动以 1080p 开始播放
3. 用户观看到 30 秒位置
4. 用户点击右上角"更多"按钮（三个点）
5. 系统显示播放选项弹窗，"清晰度"区域显示三个选项：480p、720p、1080p（1080p 为绿色高亮）
6. 用户点击"720p 高清"
7. 系统记录当前播放时间（30秒），暂停视频
8. 系统更换视频源为 720p URL
9. 系统跳转到 30 秒位置并继续播放
10. 系统显示 Toast："已切换到 720p 高清"
11. 播放选项弹窗自动关闭

**预期结果：**
✅ 视频无缝切换到 720p，从 30 秒位置继续播放
✅ 用户偏好保存到本地存储

## 术语表

| 术语 | 定义 |
|------|------|
| 分辨率 | 视频的清晰度级别，包括 480p（标清）、720p（高清）、1080p（超清） |
| 自适应降级 | 根据网络状况自动降低视频分辨率的功能 |
| 云存储 URL | 腾讯云开发云存储的文件标识符，格式为 `cloud://env-id.xxx` |
| 临时链接 | 云存储文件的临时访问 URL，有时效性 |
| 课时 | 课程中的一个视频单元或学习单元 |

## 附录

### A. 分辨率规格对照表

| 分辨率 | 中文名称 | 典型分辨率 | 带宽需求 | 推荐场景 |
|--------|---------|-----------|---------|---------|
| 480p | 标清 | 854x480 | 1-2 Mbps | 移动网络、弱网环境 |

| 4G/5G | 720p | 20 秒 |
| 3G | 480p | 15 秒 |
| 2G | 不推荐观看 | - |
