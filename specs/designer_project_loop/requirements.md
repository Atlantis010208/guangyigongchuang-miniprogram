# 需求文档 — 移除工作流步骤 & 增加验收闭环

## 介绍

当前设计师端项目详情页（`designer-project-detail`）包含一个详细的"工作流步骤"时间线（timeline），设计师需要逐步标记每个步骤完成。这套机制过于繁琐，且在设计师标记所有步骤完成后，系统直接将项目设为"已完成"，业主端无法感知也无法确认验收，导致业务流程不闭环。

本次需求：
1. **移除**设计师详情页中的"工作流步骤"时间线区块
2. **增加**设计师端"提交验收"功能
3. **增加**业主端"确认验收"功能
4. 形成 `进行中 → 待验收 → 已完成` 的完整闭环

---

## 需求

### 需求 1 — 移除工作流步骤

**用户故事：** 作为设计师，我不需要逐步标记工作流中的每个小步骤，这样操作更简洁高效。

#### 验收标准

1. 当设计师进入项目详情页时，系统**不应**显示"工作流步骤"标题及 timeline 步骤列表（圆点、步骤名、标记完成/撤销按钮）。
2. 系统**应**继续显示顶部步骤条（待确认→进行中→已完成）、项目信息、客户信息、接单时间、加急标签、底部操作按钮。
3. 系统**不应**提供 `onToggleStep` 交互能力。

### 需求 2 — 设计师提交验收

**用户故事：** 作为设计师，当我完成设计工作后，我希望通过一个按钮告知平台和客户"设计已完成，请验收"。

#### 验收标准

1. 当项目状态为 `review` 或 `design`（进行中）且 `designerConfirmed` 不为 `true` 时，系统**应**在底部操作栏显示【提交验收】主按钮。
2. 当设计师点击【提交验收】时，系统**应**弹出确认对话框："确认已完成设计并提交验收？"。
3. 当设计师确认提交后，系统**应**将项目的 `designerConfirmed` 设为 `true`、`status` 更新为 `verifying`，并记录 `verifySubmittedAt` 时间戳。
4. 当项目状态为 `verifying` 时，系统**应**在底部操作栏显示"已提交验收，等待客户确认"提示文案，替代【提交验收】按钮。
5. 当项目状态为 `done` 时，系统**不应**显示【提交验收】按钮。

### 需求 3 — 业主确认验收

**用户故事：** 作为业主，当设计师完成设计后，我希望能在小程序内查看并确认验收，使项目正式完结。

#### 验收标准

1. 当业主打开需求进度页（`progress`）且项目的 `designerConfirmed` 为 `true`、`userConfirmed` 不为 `true` 时，系统**应**在底部操作栏显示【确认验收】按钮。
2. 当业主点击【确认验收】时，系统**应**弹出确认对话框："确认已收到满意的设计方案？"。
3. 当业主确认验收后，系统**应**将项目的 `userConfirmed` 设为 `true`、`status` 更新为 `done`、记录 `completedAt` 时间戳。
4. 当项目状态变为 `done` 时，业主端顶部步骤条**应**自动推进到"已完成"阶段。
5. 系统**应**保留【联系客服】按钮不受影响。

### 需求 4 — 顶部步骤条适配新状态

**用户故事：** 作为设计师/业主，我希望顶部步骤条能准确反映项目当前所处的阶段。

#### 验收标准

1. 当项目 `status` 为 `submitted` 或 `pending` 时，步骤条**应**激活到"待确认"（index=0）。
2. 当项目 `status` 为 `review` 或 `design` 时，步骤条**应**激活到"进行中"（index=1）。
3. 当项目 `status` 为 `verifying` 时，步骤条**应**激活到"进行中"（index=1），同时顶部可显示"待验收"提示。
4. 当项目 `status` 为 `done` 或 `completed` 时，步骤条**应**激活到"已完成"（index=2）。
